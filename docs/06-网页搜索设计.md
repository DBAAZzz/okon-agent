# 网页搜索设计（基于当前代码）

## 1. 设计定位

当前项目把网页能力做成「研究子代理（research）」的一部分，而不是让主代理直接调用 `webSearch/webFetch`：

- 主代理负责路由与整合
- `research` 子代理负责检索、抓取、归纳
- 返回结构化结论和来源

## 2. 关键代码位置

- 搜索工具：`apps/agent/src/tools/web-search.ts`
- 抓取工具：`apps/agent/src/tools/web-fetch.ts`
- research 预设：`apps/agent/src/agent/subagent/presets.ts`
- 子代理执行与结果回传：`apps/agent/src/agent/subagent/index.ts`
- 主代理路由规则：`apps/agent/src/agent/prompt.ts`
- 主代理工具装配：`apps/agent/src/agent/factory.ts`

## 3. 主链路是如何触发网页搜索的

1. 系统提示词要求：遇到网页检索/事实核验/来源引用，必须调用 `research`。
2. 主代理 `ToolLoopAgent` 的 tools 中包含 `research`（由 `buildSubagentTools()` 动态生成）。
3. 当主代理调用 `research` 时，会创建一个 research 子代理。
4. research 子代理内部再按需调用：
   - `webSearchTool`（检索 URL）
   - `webFetchTool`（抓取 URL 内容）
5. 子代理把结构化结果与来源回传给主代理，主代理生成最终答复。

## 4. webSearchTool 实现细节

### 4.1 输入与 provider

`webSearchTool` 的输入参数：

- `provider`：当前枚举只允许 `tavily`（默认 tavily）
- `query`：搜索词
- `maxResults`：1~10，默认 5
- `searchDepth`：`basic | advanced`

### 4.2 实际 provider 状态

- 已实现并启用：Tavily（`searchWithTavily`）
- 预留但未启用：Brave（函数存在，调用分支被注释）

### 4.3 输出形态

成功时统一返回：

- `success/provider/query`
- `answer`（Tavily 返回）
- `results[]`（title/url/content/score/publishedDate）
- `totalResults/responseTime/requestId`

失败时返回：

- `success: false`
- `error`（含缺失 API key、HTTP 异常等）

## 5. webFetchTool 实现细节

`webFetchTool` 输入：

- `url`（必填）
- `format`：`auto|markdown|json|xml|text`
- `maxChars`：500~50000，默认 10000

执行流程：

1. `fetch(url)` 拉取原始内容（带浏览器 UA）。
2. 根据 `content-type` + `format` 决定输出格式。
3. `markdown` 模式：
   - 先用 `Readability` 抽正文
   - 再用 `Turndown` 转 markdown
   - 提取失败时尝试 fallback 到 `body` HTML
4. 文本类结果按 `maxChars` 截断并标记 `truncated`。

## 6. research 子代理的结构化约束

research 预设定义了：

- 指令：先澄清目标，再检索/抓取，禁止编造来源
- 工具集：`{ webSearchTool, webFetchTool }`
- 最大步数：`defaultMaxSteps = 4`
- 输出 schema：
  - `conclusion`
  - `keyFindings[]`
  - `sources[]`
  - `uncertainties[]`

`buildSubagentTools()` 会把该子代理包装成主代理可调用的一个 tool（名称为 `research`）。

## 7. 结果如何回到主代理

`toSubagentResult()` 会把子代理执行过程转成：

- `output`：结构化结果（若 schema 解析成功）
- `text`：兜底文本
- `steps`：工具调用轨迹
- `sources`：从工具输出中递归提取到的 URL

`toModelOutput()` 再序列化为文本，附加 `Verified Sources`，作为 `research` 工具结果返给主代理。

## 8. 当前能力边界

- `webSearchTool` / `webFetchTool` 均 `needsApproval: false`（默认自动执行）
- provider 当前实质上只有 Tavily
- 来源可信度依赖外部搜索与网页内容质量，项目内暂无额外事实校验层
