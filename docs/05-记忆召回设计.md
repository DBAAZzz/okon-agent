# 记忆召回设计（基于当前代码）

## 1. 目标与边界

当前实现是「会话内短期记忆增强」，不是完整 RAG：

- 存储介质：Qdrant 集合 `memories`
- 向量形态：仅稀疏向量（`bm25`），未接入 dense embedding
- 召回策略：主流程默认使用「最近记忆」而非相似度搜索
- 作用域：按 `sessionId` 过滤，只在当前会话生效

## 2. 关键代码位置

- 初始化入口：`apps/agent/src/server.ts`
- 记忆能力导出：`apps/agent/src/capabilities/memory/index.ts`
- 存取实现：`apps/agent/src/capabilities/memory/memory-store.ts`
- 稀疏向量转换：`apps/agent/src/utils/sparse-vector.ts`
- 分词与 BM25 基础：`apps/agent/src/utils/BM25Index.ts`
- 注入主代理提示词：`apps/agent/src/agent/gateway.ts` + `apps/agent/src/agent/prompt.ts`

## 3. 数据结构

记忆结构定义在 `apps/agent/src/capabilities/memory/types.ts`：

- `MemoryFilter`：至少包含 `sessionId`，并允许额外字符串维度
- `MemoryPayload`：`content` + `filter` + `createdAt`
- `MemorySearchResult`：含 `id/score`，用于召回结果

## 4. 写入流程

### 4.1 集合与索引初始化

`createMemoryStore()` 首次操作时会执行 `ensureCollection()`：

1. 检查 `memories` 是否存在，不存在则创建。
2. 创建稀疏向量配置：`sparse_vectors.bm25.modifier = idf`。
3. 尝试创建 `createdAt` payload 索引（`datetime`），用于按时间排序。

### 4.2 对话写入

主流程在 `finalizeStream()` 的「无待审批」分支里异步调用：

- `memoryStore.storeConversation(userMessage, response.messages, { sessionId })`

`storeConversation()` 的行为：

1. 仅提取 assistant 的文本内容（字符串内容或 `part.type === 'text'`）。
2. 组装为一条文本：`用户: ...\n助手: ...`
3. 调用 `add()` 写入 Qdrant，向量由 `textToSparseVector()` 生成。

说明：如果 assistant 没有可提取文本，则跳过写入。

## 5. 召回流程

### 5.1 主流程实际使用：recent

`runAgent()` 在收到新用户消息时调用：

- `memoryStore.recent({ sessionId }, 3)`

即按 `createdAt desc` 取最近 3 条会话记忆，不做语义相似度排序。

### 5.2 可用但未接入主流程：search

`memory-store.ts` 还实现了 `search(query, filter, limit)`：

- 使用 BM25 稀疏向量做 Qdrant `search`
- 支持 filter 映射到 `filter.xxx` payload 条件

但当前主代理链路未调用该接口。

## 6. 提示词注入

`runAgent()` 把召回结果映射成文本数组后传给 `buildSystemPrompt()`：

- `buildSystemPrompt({ memories: memories.map(m => m.content) })`

`buildSystemPrompt()` 会在系统提示词末尾追加：

- `## 相关记忆`
- 每条记忆一行拼接

这使主代理在本轮推理时可直接读取近期上下文。

## 7. 当前实现特点与限制

- 优点：
  - 结构简单，和 ToolLoopAgent 主链路耦合低
  - Qdrant 稀疏检索成本低，中文分词与关键词召回稳定
- 限制：
  - 只做会话内记忆，没有跨会话用户画像
  - 主流程未使用相似度召回，仅按时间最近 3 条注入
  - 记忆内容是原始对话拼接，未做摘要/实体化/衰减策略
